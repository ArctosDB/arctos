Part Locations

Part locations are stored in a hierarchical fashion using a recursive model. The following example places a tissue sample in a nunc tube in a box in a rack in a freezer. The freezer is a root container - it's location is not tracked (and assumed to be easy enough to find that it doesn't need tracking). The tissue sample is a leaf container - it is the smallest object tracked in this example.
Container ID Parent Container ID Details
1 0 freezer
2 1 rack
3 2 box
4 3 nunc tube
5 4 tissue sample

Searching for part or container locations will return the item(s) you searched on and their parent hierarchy. Use the See all containers in this container or See all collection objects in this container links to examine contents.

See all containers in this container will display all containers that are direct children of the selected container.

See all collection objects in this container will display all leaf containers that are:

  1. Collection Objects (ie, parts)
  2. Children (not necessarily direct children) of the selected container
